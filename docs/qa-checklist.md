# QA Checklist

## Execution Readiness
- Test environment is available and stable
- Browser/device matrix is confirmed
- Test data and user accounts are prepared
- Entry criteria are met

## Test Case Quality
- Each test case has clear preconditions
- Steps are reproducible
- Expected results are explicit
- Priority and severity are assigned

## Defect Quality
- Each defect has severity, priority, environment, and reproducible steps
- Expected and actual results are clearly separated
- Evidence path or recording reference is included where applicable
- Business impact is documented for Critical and Major issues

## Release Readiness
- RTM has no unmapped high-priority requirements
- Critical defects are reviewed
- Test summary report includes pass/fail/block counts
- No draft notes remain in public-facing artifacts
